# Game Design Document (Turn-based War Strategy)

## 1) 产品定位
- 题材：现代战争 / 区域冲突 / 战役推演
- 视角：战略层（资源、部署）+ 战术层（战斗执行）
- 玩家目标：在限定回合内达成战役胜利条件

## 2) 核心回合循环
1. **情报阶段**：更新侦察、视野、已知敌情
2. **指令阶段**：移动、攻击、补给、建造、外交行动
3. **结算阶段**：战斗、占领、士气、资源与事件结算
4. **回合结束**：胜负检查、事件推进、存档/回放节点

## 3) 胜负与计分
- 胜利条件（可组合）：
  - 控制关键城市/港口/空军基地 N 回合
  - 敌方战力跌破阈值
  - 达成战役目标分数
- 失败条件：
  - 己方关键据点全部失守
  - 战略资源（补给/资金）崩溃
  - 超过回合上限仍未达成目标

## 4) 关键系统
- 单位系统：机动、火力、装甲、射程、补给消耗、维护
- 地图系统：地形加成、控制区（ZOC）、道路/海运/空运线路
- 战斗系统：命中、穿甲、压制、士气、地形修正
- 补给系统：补给线长度、节点容量、中断惩罚
- 外交系统：停火、援助、制裁、国际舆论影响
- AI 系统：战略目标选择 + 战术行动评分

## 5) 平衡原则
- 强调“信息与补给”对战局的决定性作用
- 避免单一兵种无脑碾压（反制链必须成立）
- 控制滚雪球效应，给劣势方保留翻盘窗口

## 6) 版本里程碑
- M1：单地图 + 基础回合循环 + 战斗结算
- M2：补给与士气系统 + AI 首版
- M3：战役目标、多剧本、平衡迭代
