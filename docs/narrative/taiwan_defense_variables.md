# 台湾防御关键变量

本文件用于说明台湾阵营真正依赖哪些变量撑住战局，并给系统设计提供映射方向。

| 变量 | 战争意义 | 游戏映射建议 |
|---|---|---|
| 地理与登陆窗口 | 可用滩头、港口、机场与展开轴有限，防守方更容易集中阻绝。 | 让地图节点、天气与海况直接影响敌方升级行动成本。 |
| 指挥通信存活 | 雷达、通信、数据链与指挥冗余决定部队能否持续协同。 | 影响行动可用性、反应速度与事件失败率。 |
| 港口与交通节点 | 港口、机场、桥梁和隧道决定机动、补给与城市防御节奏。 | 作为关键设施状态条，直接牵动战备与防御效率。 |
| 封锁承受力 | 能源、贸易与外部补给高度依赖海空通道。 | 映射为经济压力、补给短缺与社会韧性消耗。 |
| 弹药与维修持续性 | 能否持续补充导弹、油料、零件与抢修能力，决定能撑多久。 | 影响拒止火力恢复、行动次数与高强度作战衰减。 |
| 社会韧性与民心 | 恐慌、认知战、避难秩序与地方协同会直接反噬战力。 | 作为独立韧性指标，联动事件池、动员速度与内政惩罚。 |
| 外援接入效率 | 情报、备件、通信与军援速度会显著改变防御曲线。 | 通过外援进度条或事件链影响恢复、侦察与高级装备可用性。 |

## 设计结论
- 台湾阵营不应只按兵力总量建模，而应强调拒止火力存活率、动员速度与基础设施韧性。
- 若玩家未来可切换阵营，台湾侧更适合围绕“撑住多少天”和“保住多少关键节点”设计回合压力。
- 若当前原型继续以中方为玩家，台湾侧变量应主要体现在对手稳定、恢复能力与事件反制质量上。
