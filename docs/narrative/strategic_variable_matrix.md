# 战争关键变量矩阵

本文件用于说明哪些因素真正决定战局，并给系统设计提供映射方向。

| 变量 | 战争意义 | 游戏映射建议 |
|---|---|---|
| 作战目标 | 惩罚、封锁、威慑与占领对应完全不同的成本结构。 | 用阶段目标或终局权重区分“有限施压”与“夺岛升级”。 |
| 后勤与跨海运力 | 能否持续送上岸、保住港口、维持补给线，比首波突击更关键。 | 影响战备恢复、升级行动门槛与后续回合衰减。 |
| 第三方介入速度 | 外部护航、制裁与军事部署会改变风险和对手韧性。 | 直接挂钩国际风险、事件池与对手稳定恢复。 |
| 天气与海况窗口 | 封控和无人战可高频出现，全面登陆则更依赖窗口期。 | 为海空行动设置天气修正，尤其限制高阶登陆行动。 |
| 经济承受力 | 封控会反噬贸易、能源与国内社会承压。 | 主要映射为内压增长、恢复效率下降与经济事件链。 |
| 联合作战执行质量 | 编制存在不等于能在高损耗实战中顺畅运转。 | 作为高强度行动的失败概率、摩擦事件与收益波动来源。 |

## 设计结论
- 兵力表只定义“能做什么”，关键变量才决定“能不能做成”。
- 若游戏主线是有限施压，重点应放在封控、风险管理与耐力竞争。
- 若游戏后续扩展到登陆战，必须先补齐海运、港口、天气与补给规则。
